Purpose

The purpose of this application is to explore the design of an Angular Signal-based application with a focus on maintaining sustainable development velocity and facilitating the seamless addition of new features over time. This is achieved through leveraging modern Angular capabilities, such as Angular Signal, Deferrable Views and Control Flow syntax. All components are fully driven by Signals. Furthermore, the project includes a custom reactive state management system built on Signals, ensuring efficient and predictable state handling.

This project represents a migration from a Component-First architecture (standalone) to a Signal-based Component Architecture, utilizing Angular CLI 19. This application present high-level recommendations for a well-designed Angular application architecture based on best practices and battle-tested patterns.

- Signal-based Component Architecture.
- Smart and Dumb components pattern.
- Reactive state management (using own Signal store).

Extras:

- Dynamically load a configuration file before the app starts.
- Register and use custom SVG icons within the application
- Implement internationalization using a third-party library.
- Integrate Infinite Scroll to a view listing.
- Implement Global Error Handling in the web application.
- Setting Page Titles Natively With The Angular Router.
 Description

This application is built using Angular 19 and own Signal store as state management. The design was created using Angular Material, and unit tests were conducted with Jasmine and Karma. The application provides the following features:

- Multilingual Support (Internationalization in 3 Languages).
- Apartment Listings Page.
- Filter Apartments by City and Borough, and Load More Data via Scroll.
- Highlighting Favorite Apartments.
- Apartment detail page.
- Saving Apartment as Favorites.
- Apartment favourites page.
